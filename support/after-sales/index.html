售前 ​

  • 模型选择
  • 价格和充值方式
  • 业务、对公转账和开票：暂不支持

售后 ​

  • Key 无法使用
  • 账单或余额问题
  • 客户端接入异常
  • 登录、重置密码、邮箱相关问题

还没做过最小自检时，建议先看一次 排障手册。很多问题在本地先确认一遍，会比直接描述“不能用”更容易定位。

什么时候先自检，什么时候直接联系 ​

更适合先自己查一轮的情况：

  • 工具刚配置完，还没做过最小 curl 测试
  • 不确定自己把地址写成根地址还是 /v1
  • 模型名是凭印象手打的，不是从模型广场复制的
  • 改了环境变量，但没有重新打开终端

更适合直接联系的情况：

  • 余额、扣费、订单异常；业务、对公转账和开票暂不支持
  • 登录、邮箱、密码重置存在问题
  • Key 已确认无误，但控制台状态和账单表现异常
  • 已经做完最小自检，仍然无法定位问题

联系前建议先准备 ​

  • 账号邮箱
  • 报错截图
  • 所用工具名
  • 关键配置截图（记得打码 Key）
  • 模型名
  • 最小 curl 测试结果
  • 发生问题的大概时间
  • 操作系统版本

信息越完整，定位越快。

什么信息最关键 ​

真正最有助于定位的，通常不是一张“报错了”的大截图，而是下面这几项能对应起来：

  • 你用的工具名称
  • 你填的 Base URL
  • 你复制的模型名
  • 报错原文或错误码
  • 最小 curl 请求结果

只要这几项能对上，售后通常就能很快判断你卡在安装层、配置层，还是网关层。

截图怎么打码 ​

需要打码：

  • API Key 中间和尾部
  • 邮箱的部分字符
  • 订单号中间部分
  • 服务器 IP 或内部域名

可以保留：

  • Base URL
  • 模型名
  • 错误码
  • 报错原文

如果是配置页截图，建议把“地址、模型、错误提示”一起保留在同一张图里。
这样售后能更快判断你卡在地址层、模型层还是鉴权层。

常见沟通误区 ​

  • 只说“不能用”，不带工具名、报错原文和模型名
  • 把完整 Key 直接发出来
  • 截图只截到报错弹窗，没截到当前地址和模型
  • 明明还没做最小测试，就直接怀疑网关整体故障

把这些误区避开以后，定位速度通常会快很多。
