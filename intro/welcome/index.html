欢迎使用 小蓝中转站 ​

小蓝中转站 是专为开发者打造的统一 AI 编程模型网关。在这里，你不再需要为网络连通性、跨国支付壁垒和繁琐的多平台账号管理而头疼。

一个入口，一个账号，统一管理模型、Key、账单和常用开发工具接入。

如果你刚开始接触 Claude Code、Codex 或 Cherry Studio，先从“新手上路”开始；如果已经熟悉 OpenAI 兼容接口，直接复制 Endpoint、Key 和模型名即可接入。

为什么是 小蓝中转站？ ​

🔌 统一接口接入 ​

在支持 OpenAI 兼容接口的工具中填入 https://xiaolan1.cn/v1 和你的专属 Key，即可通过统一网关调用可用模型。

但第一次接入时，最好再多记一句：
不是所有工具都直接填 https://xiaolan1.cn/v1。Claude Code、Gemini CLI、Cherry Studio 这类工具还有各自的地址口径，文档里都已经单独拆开说明。

💳 人民币余额与账单 ​

站内余额和消费记录按人民币展示。模型价格与倍率在模型广场查看，充值和请求明细在控制台查看。

🛠️ 开发者工具优先 ​

文档覆盖 Claude Code、Codex、Gemini CLI、Cherry Studio 等常用工具，包含下载入口、基础环境、配置项和最小验证步骤。

🔐 聚合且安全的凭证中心 ​

在 小蓝中转站 控制台可以按设备、工具或项目创建独立 Key，配置额度、权限和启用状态，便于定位消耗和处理泄露。

先看懂四个最常用的概念 ​

第一次接入时，最容易混淆的不是某个命令，而是这四件事到底各自负责什么：

概念作用用在哪里
账号登录控制台、查看余额、日志和订单官网控制台
余额支付模型请求费用钱包 / 账单
Key客户端调用网关时的凭证CLI、桌面客户端、脚本
模型实际请求的目标模型model 字段

只要其中一项没准备好，客户端就可能报错。所以第一次接入时，先把顺序走通，比一开始就换模型、换工具更重要。

推荐接入顺序 ​

第一次配置时按此顺序完成。每一步都对应一个常见卡点：本地命令、账号、Key、模型名和工具配置。

  1. 🧭 Windows 快速接入 / macOS 快速接入：完成本地环境和 CLI 配置。
  2. 🏁 注册账号：登录控制台。
  3. 🔑 创建专属 Key：生成工具调用凭证。
  4. 🚀 配置编程工具：以 Claude Code 为例，也可以直接选择左侧对应工具。

第一次最容易卡住的，不是哪一个命令，而是顺序乱了：

  • 还没充值，就先怀疑模型名错了
  • 还没建 Key，就先怀疑工具安装坏了
  • 还没分清根地址和 /v1，就先到处改环境变量

按这四步顺着走，通常能少掉很多反复回退。

第一次接入怎么选工具 ​

不需要把所有工具都装一遍。先选一条最贴近当前目标的路径，通常最容易一次跑通。

你现在要做什么建议先用什么原因
先确认接口是否可用Cherry Studio图形界面最直观，先验证地址、Key 和模型
想直接在项目目录里改代码Claude Code终端工作流最直接
已经熟悉 OpenAI CLI 风格Codex配置方式更接近 OpenAI 兼容用法
想做长文总结或大仓库理解Gemini CLI更常见于长上下文场景

一旦先跑通其中一个工具，后面再配置第二个工具时，就能把问题范围缩小到“工具差异”，而不是从头怀疑账号、Key、余额和模型。

真正算“第一条链路跑通”时，通常要同时满足：

  1. 工具能正常启动
  2. 发一句短消息后能正常返回
  3. 控制台日志里能看到对应请求
  4. 你知道当前这条请求用的是哪一把 Key、哪个模型
