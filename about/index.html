关于 小蓝中转站 ​

小蓝中转站 面向日常编码场景提供统一接入入口。

  • 官网：https://xiaolan1.cn
  • 文档：https://docs.xiaolan1.cn
  • 模型广场：https://xiaolan1.cn/pricing
  • 站长 QQ：1527997732

如果你是第一次接入，最省时间的路径通常是：

  1. 先看 快速接入
  2. 注册账号并充值
  3. 创建一把专用 Key
  4. 选一个最想用的工具先跑通

先跑通一条最小链路，再去扩展第二个工具，通常会比一开始同时配置多个客户端更稳。

很多人第一次接入时，会把这三件事混在一起：

  • 工具怎么下载
  • Key 在哪里创建
  • 地址、模型和 Key 该怎么填

小蓝中转站 主要解决的是后两件事。
工具本身仍然优先从各自官方入口下载；真正接模型时，再通过 小蓝中转站 把地址、Key、模型和账单统一起来。

适合谁 ​

  • 想用 Claude Code、Codex、Gemini CLI 的开发者
  • 想用 Cherry Studio 管理多模型的用户
  • 需要国内直连 AI 接口的团队
  • 想把账单、Key、模型统一管理的人

你会在这里统一管理什么 ​

大多数用户最终会把这几件事都集中到同一个控制台里处理：

  • 账号和余额
  • 每个工具使用的 Key
  • 当前可用模型和价格
  • 请求日志和异常排查

这也是 小蓝中转站 和“只提供一个 API 地址”的区别。它不仅是一个入口，还把后续最常用的管理动作一起收进了控制台。

真正用起来以后，最常见的日常动作通常就是这几类：

  • 新装一个工具后，回来创建一把专属 Key
  • 想换模型时，回模型广场复制模型名
  • 遇到报错时，回日志里按 Key 查请求
  • 想控制预算时，回令牌管理里改额度或模型权限

常用入口 ​

  • API Endpoint：https://xiaolan1.cn/v1
  • 注册与控制台：https://xiaolan1.cn
  • 模型价格：https://xiaolan1.cn/pricing
  • 问题排查：常见问题
  • 新手路线：Windows 快速接入 / macOS 快速接入
