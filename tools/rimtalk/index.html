RimTalk 使用与填写教程 ​

RimTalk 是环世界 / RimWorld 里把角色对话接到 AI 的工具。接入 小蓝中转站 时，优先选择 Custom 或 OpenAI Compatible 这一类自定义服务商。

这类工具最容易填错的是地址。RimTalk 的 Custom 配置里，通常先填根地址：

txt
https://xiaolan1.cn
1

模型第一次测试建议填：

txt
gpt-5.4-mini
1

先按这一版填

如果你在 RimTalk 里选的是 Custom，地址先填 https://xiaolan1.cn。不要先填 /v1，也不要先选 Gemini，除非你的模型列表里已经明确能看到 Gemini 模型并且这把 Key 有权限。

先填这一组 ​

RimTalk 里的字段填写内容
Provider / ModeCustom 或 OpenAI Compatible
Base URL / API URL / Root URLhttps://xiaolan1.cn
API Key控制台创建的 sk-... 密钥
Modelgpt-5.4-mini
Temperature保持默认，或先填 0.7
Max Tokens保持默认即可；如果必须填写，先用 600 到 1000

Key 建议单独给 RimTalk 建一把，名字可以写成 RimTalk-PC。后面看日志、查余额、停用 Key 都更清楚。

字段对照 ​

不同版本 RimTalk 的字段名可能不一样。按字段含义来填：

如果字段写的是应该填
Base URL / API URL / Root URLhttps://xiaolan1.cn
OpenAI Base URLhttps://xiaolan1.cn/v1
Chat Endpoint / API Endpointhttps://xiaolan1.cn/v1/chat/completions
Models Endpointhttps://xiaolan1.cn/v1/models
Model Namegpt-5.4-mini，或从模型广场复制的完整模型名

如果同一个页面只有一个 URL 输入框，就先看它的标签。写着 Base URL、API URL、Root URL 时填根地址；明确写着 Endpoint 时才填完整接口地址。

为什么 Custom 里优先填根地址 ​

RimTalk 的 Custom/OpenAI 客户端通常会把你填的根地址继续拼成 /v1/chat/completions 请求。也就是说，你填根地址时，它实际请求的是：

txt
https://xiaolan1.cn/v1/chat/completions
1

如果把 Base URL 填成 https://xiaolan1.cn/v1，有些版本会把请求发到错误路径，结果就是 404；如果填了完整接口但字段本身又会自动拼路径，也可能重复拼接。

判断规则很简单：Base URL 填根地址，Endpoint 填完整接口。

第一次测试 ​

  1. 在 RimTalk 保存 Custom 配置。
  2. 先进一个小存档，触发一句很短的角色对话。
  3. 打开 RimTalk 的 API Log / Debug Log。
  4. 确认请求地址最终包含 /v1/chat/completions。
  5. 确认模型名是 gpt-5.4-mini，Key 是你新建的那把。

第一次只测一条短消息。能正常返回之后，再换更贵、更强或上下文更长的模型。

常见报错 ​

现象常见原因怎么处理
400模型名写错、Key 没权限、请求体格式不符合当前模型模型先改成 gpt-5.4-mini；模型名尽量从模型广场复制
404URL 路径填错，最常见是 Base URL 填了 /v1Custom 里的 Base URL 改成 https://xiaolan1.cn
503上游繁忙、当前模型暂不可用，或实际请求地址和你以为的不一样先换 gpt-5.4-mini 重试；再看 API Log 里的真实请求地址
模型列表为空Key 权限不足，或 Models Endpoint 填错手动填模型名；需要模型列表时用 https://xiaolan1.cn/v1/models
没有任何反应RimTalk 没读到新配置，或游戏里没有触发到请求保存后重启游戏 / mod，先用最短对话触发一次

拿日志给别人排查 ​

如果还是不通，不要只发“400 / 404 / 503”。优先复制这三样：

  1. API Log 里 API request: 后面的真实 URL。
  2. 请求 JSON 里的 model 字段。
  3. 错误返回内容。

发给别人前，把 Authorization、Bearer sk-... 和完整 Key 打码。

接口自测 ​

如果想先确认 Key 本身能不能用，可以在 Windows PowerShell 里测模型列表：

powershell
curl.exe https://xiaolan1.cn/v1/models -H "Authorization: Bearer sk-你的Key"
1

再测一次最小聊天请求：

powershell
curl.exe https://xiaolan1.cn/v1/chat/completions -H "Content-Type: application/json" -H "Authorization: Bearer sk-你的Key" -d "{\"model\":\"gpt-5.4-mini\",\"messages\":[{\"role\":\"user\",\"content\":\"test\"}]}"
1

这两步能通，说明 Key 和小蓝接口本身是通的；RimTalk 里再重点看 URL 字段、模型名和 Custom 模式。
