Claude Code Hub ​

Claude Code Hub 适合把 Claude Code 的配置集中起来管理，尤其适合多设备或多人协作环境。

它更像一个“面向 Claude Code 的统一配置入口”，而不是 Claude Code 本体。也就是说：

  • Claude Code 仍然要先能正常安装和运行
  • Claude Code Hub 负责把地址、Key、模型和环境配置集中起来管理

Claude Code 本地还没跑通时，先看 Claude Code 部署。

个人单机使用时，通常不需要先上 Claude Code Hub。
一台机器只配自己用时，直接按 Claude Code 部署 或用 CC-Switch 往往更省事。

第一次接入时，先把这一点分清楚会省很多时间：

  • Claude Code：真正执行对话、读代码、改文件的 CLI
  • Claude Code Hub：帮你集中管理 Claude Code 相关配置和入口

也就是说，Hub 解决的是“怎么更统一地管理”，不是“替代 Claude Code 本体”。

工具介绍 ​

它负责把 Claude Code 相关配置集中起来管理：

  • 接口地址
  • API Key
  • 模型选择
  • 常用运行环境

它更适合解决这类问题：

  • 多台机器想复用一套 Claude Code 配置
  • 多个人想统一切换同一批模型和环境
  • 不想每次换设备都手写同一批环境变量

适合什么场景 ​

  • 需要多人复用同一套配置模板
  • 一台机器要切多种模型
  • 想把 CLI 配置集中管理
  • 不想每台设备都手动写一遍环境变量

不太适合的场景：

  • 只是单机自用，先想把第一条链路跑通
  • 还没搞清楚 Claude Code 根地址和 /v1 的区别
  • 还没跑通过一把最小可用的 Claude 专用 Key

快速部署 ​

部署 ​

按官方项目的安装说明完成部署后，进入管理面板添加自己的配置。

客户端接入 ​

新建配置时，填写：

  • Base URL：https://xiaolan1.cn
  • API Key：你的 Claude 专用 Key
  • 模型：从模型广场复制 Claude 系列模型名

Claude Code / Claude Code Hub 使用 Claude 协议根地址，这里不要加 /v1。Codex 这类直接填写 OpenAI Base URL 的工具才通常填写 https://xiaolan1.cn/v1。Cherry Studio 的服务商地址栏则更适合填写根地址 https://xiaolan1.cn，再由客户端自己补兼容路径。Gemini CLI 当前版本使用官方 Gemini 变量时，也填根地址，再由 CLI 自己拼接 Gemini API 路径。

第一次接入时，建议先只配一条最小配置：

  1. 一把专门给 Claude Code 用的 Key
  2. 一个已经确认可用的 Claude 系列模型
  3. 一个最简单的测试项目目录

先确保最小配置能正常返回，再继续拆主力环境、测试环境和多人模板。

多人环境里使用时，第一次也建议先只接一台机器、一个模型、一个 Key。
这样你能先证明 Hub 到 Claude Code 这一层已经通了，再去扩展第二台设备和更多模板，排障会轻很多。

真正算接通时，通常要同时满足：

  1. Claude Code Hub 里保存的配置能正常应用
  2. Claude Code 启动后没有跳回官方登录流程
  3. /model 或当前会话能读到你刚配置的模型
  4. 发一句短消息后能正常返回

如果这里只有第 1 步成功，第 2 到第 4 步都不对，优先不要急着改 Hub 结构。
先回头看：

  • Claude Code 本机是不是本来就没跑通
  • 当前 Base URL 是否误填成了 /v1
  • 这把 Key 的分组和模型权限是否真的允许当前模型

使用建议 ​

  • 主力环境和测试环境分开建
  • 不同设备用不同 Key
  • 配好后先跑一次最小对话，确认能正常返回
  • 配置名尽量写清楚设备或用途，后续更好排查

常见问题 ​

和直接配环境变量有什么区别 ​

环境变量更轻，Claude Code Hub 更适合集中管理和切换。

可以简单理解成：

  • 环境变量：适合个人、本机、先跑通
  • Claude Code Hub：适合多设备、多人、集中管理

为什么模型列表不对 ​

通常是这把 Key 的可见模型范围不同。回控制台或模型广场核对。

如果保存配置后还是沿用旧模型，先看两件事：

  1. 当前会话是不是已经重新读取了刚保存的配置
  2. 这把 Key 的分组和模型权限里，是不是真的放行了该模型

保存成功，但 Claude Code 还是沿用旧配置 ​

先不要急着重建整套 Hub。优先检查：

  1. 当前 Claude Code 会话是不是还没重新读取配置
  2. 本机环境变量里是否还留着旧的 ANTHROPIC_BASE_URL 或 ANTHROPIC_AUTH_TOKEN
  3. 当前机器到底接的是 Hub 管理的配置，还是另一套本地手写配置

为什么 Claude Code Hub 里填了 /v1 反而不通 ​

因为 Claude Code / Claude Code Hub 走的是 Claude 协议根地址，不是普通 OpenAI 兼容客户端的 /v1 地址。这里应填写：

txt
https://xiaolan1.cn
1
