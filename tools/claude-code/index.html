Claude Code ​

Claude Code 适合在项目目录里直接读文件、改代码和查看差异。

如果你要把 Claude Code 接到 小蓝中转站，优先看：Claude Code 部署。

为什么 Claude Code 前面总会先提 Git、Node.js 和 npm ​

很多新手第一次看这类文档时，会觉得“我只是想用 AI，为什么还要先装 Git”。

原因其实很直接：

  • Git 不是为了强迫你提交代码，而是为了让 Claude Code 更稳地看懂仓库状态、差异和改动范围
  • Node.js 和 npm 仍然是很多 CLI 工具的基础环境，后面继续装 Codex、Gemini CLI 或其他 npm CLI 时也会复用

不过现在的 Claude Code 不只支持 npm。
官方当前还支持原生安装脚本、Homebrew 和 WinGet，所以“想先把 Claude Code 跑起来”这件事，本身已经不一定非要先装 npm。

如果你后面还要继续折腾 Codex、Gemini CLI 或其他命令行工具，Node.js 这套环境还是建议准备好；如果你只是先想让 Claude Code 单独跑起来，原生安装脚本、Homebrew 或 WinGet 也可以先走通。

Claude Code 最关键的一点 ​

Claude Code 填的是 Anthropic 协议根地址：

txt
https://xiaolan1.cn
1

不要手动补 /v1。
Claude Code 会自己继续拼 Claude 协议需要的路径。如果你手动补成 https://xiaolan1.cn/v1，请求很容易变成重复路径，最后看起来像“Key 失效”或“接口异常”。

第一次接入时，先记住这三项：

txt
ANTHROPIC_BASE_URL=https://xiaolan1.cn
ANTHROPIC_AUTH_TOKEN=sk-你的专属Key
CLAUDE_CODE_ENABLE_GATEWAY_MODEL_DISCOVERY=1
1
2
3

其中：

  • ANTHROPIC_BASE_URL 决定请求发往哪个网关
  • ANTHROPIC_AUTH_TOKEN 决定请求使用哪一把 Key
  • CLAUDE_CODE_ENABLE_GATEWAY_MODEL_DISCOVERY=1 允许 Claude Code 向网关读取可用模型列表

第一次接入前先准备 ​

  • 一把给 Claude Code 单独使用的 Key，例如 Windows-Claude（侧边栏 API 密钥 创建）
  • 一个准备先测试的模型（从模型广场复制）
  • 一个可用终端
  • Node.js、npm、Git 都已经能正常运行

API 密钥入口

登录控制台后点左侧 API 密钥，再点页面左上角 创建 API 密钥。如果列表为空也正常，直接创建即可。

Key 单独拆出来的意义，在于后面查日志、吊销配置和限制额度都会更清楚。

如果你还没决定装法，可以按这个思路选：

  • 想尽快开始：原生安装脚本
  • macOS 常用 Homebrew：Homebrew
  • Windows 常用系统包管理：WinGet
  • 后面还要继续装其他 npm CLI：npm

模型切换常见有三种方式：

  • claude --model ...：只影响这一次启动
  • /model：进入 Claude Code 后切换
  • ANTHROPIC_MODEL：给终端环境设置默认模型

第一次成功的判断 ​

第一次配置完成后，建议先在空目录里做最小测试。真正算成功时，通常要同时满足：

  1. claude 命令能正常启动
  2. 没有跳回官方登录流程
  3. 发一句短消息后能正常返回
  4. 你确认当前读到的是 小蓝中转站 的地址和 Key
  5. 当前模型确实是你刚配置或刚切换的那一个

先把这一步跑通，再切到真实项目，排障会轻松很多。

继续配置 ​

  • 详细安装与环境变量写法： Claude Code 部署
  • 还没装命令行环境： Windows 快速接入 / macOS 快速接入
  • 想用图形界面管理配置： CC-Switch
