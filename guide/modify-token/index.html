修改令牌设置 ​

令牌创建后仍可在控制台调整启用状态、额度、模型权限和有效期。

修改后会影响后续请求。已经发出的请求可能不会立即停止。

修改令牌的意义，不只是“把某个字段改一下”，而是让同一把 Key 的可用范围更贴近当前场景。比如：临时测试结束后收紧额度、给脚本单独限制模型、怀疑泄露时先禁用。

可调整内容 ​

登录控制台，进入 令牌管理，点击令牌右侧的编辑按钮。

修改前先想清楚你现在要解决的是哪一类问题：

  • 只是想限制预算
  • 只是想限制模型范围
  • 只是想临时停用
  • 还是这把 Key 的使用对象已经变了

把问题先分清楚，才容易判断应该“编辑原 Key”还是“直接新建一把”。

1. 额度控制 ​

  • 批量脚本、自动化任务和共享测试 Key 建议设置总配额。
  • 团队测试 Key 可设置周期额度。
  • 达到额度后，网关会拒绝该 Key 的后续请求。

额度更适合解决这类问题：

  • 担心脚本误循环把余额快速打空
  • 想给测试同学或外包同事一个可控上限
  • 想把桌面聊天和 CLI 编程的预算分开

2. 启用或禁用 ​

遇到以下情况，先禁用对应 Key：

  • 怀疑项目的 .env 文件被意外 Push 到了公共仓库。
  • 离职交接或外包项目结束。
  • 某个工具产生异常高频请求。

禁用通常会很快生效。长连接或已经发出的请求可能需要等待结束。

禁用更适合“先止血，再细查”。
一旦怀疑泄露或发现异常高频请求，先禁用，比一边怀疑一边继续让请求跑更稳。

3. 模型权限 ​

可以修改 Key 绑定的模型权限组。只做文本整理、翻译或轻量问答的工具，可限制为低成本模型；代码生成和长上下文任务再开放强模型。

模型权限更适合解决：

  • 某个工具不应该碰高价模型
  • 某个共享 Key 只允许调用一小批模型
  • 某台服务器只允许跑脚本专用模型

什么时候改这把 Key，什么时候直接新建 ​

不是所有情况都适合继续修改原有 Key。可以按这个思路判断：

场景更适合怎么做原因
只是临时调小额度、调整有效期直接编辑当前 Key原工具配置不用重写
想给同一台设备补充几个可用模型直接编辑当前 Key对现有工作流影响最小
想把 Claude Code 和 Cherry Studio 的消耗分开新建一把 Key后续日志更清楚
怀疑 Key 已泄露新建 Key，并禁用旧 Key比继续编辑旧 Key 更稳
给同事、脚本或服务器单独使用新建独立 Key便于回收和限额

简单说：

  • 只改规则，不改身份：优先编辑
  • 换使用对象、换安全边界：优先新建

常见调整场景 ​

  • 临时测试：设置较小总额度，测试完直接禁用。
  • 桌面聊天：允许常用聊天模型，额度适中。
  • 终端编程：允许代码能力强的模型，单独统计消耗。
  • 服务器脚本：必须设置额度上限，避免循环请求。
  • 共享给他人：设置过期时间和模型范围。

如果你现在拿不准，先按这个最简单的判断走：

  • 还想继续让原工具用同一把 Key：先编辑
  • 想把不同人、不同机器、不同工具分开：直接新建

修改后多久生效 ​

禁用、额度、权限变更通常会很快影响后续请求。已经发出的请求不一定会立刻停止，所以发现泄露时建议直接禁用或删除 Key。

如果你刚修改完，最容易误判的一点是：
控制台页面已经改了，不代表当前工具会话一定已经读到了新的结果。
桌面客户端、长会话 CLI、后台服务，都可能还在延续旧会话或旧配置。

修改后怎么确认已经生效 ​

最稳的验证方式不是只看控制台，而是做一次最小请求：

  1. 修改 Key 设置
  2. 用这把 Key 再发一条最小请求
  3. 看返回是否符合预期
  4. 回控制台日志确认请求记录

例如：

  • 刚禁用 Key：请求应直接失败
  • 刚去掉某个模型权限：该模型应不再可调用
  • 刚放宽额度：原本被限额的请求应恢复

这样能把“页面上已经改了”和“客户端实际已经生效”区分开。

如果测试结果和预期不一致，再回头看三层：

  1. 当前工具是不是还在用这把 Key
  2. 当前会话是不是还停留在旧模型
  3. 当前服务或终端是不是需要重新打开

Key 泄露处理 ​

  1. 立即禁用或删除这把 Key。
  2. 检查日志，看最近请求是否异常。
  3. 给对应工具创建一把新 Key。
  4. 更新本地配置。
  5. 确认 Git 仓库里没有残留明文 Key。

备用 Key ​

常用开发环境可以保留一把备用 Key。主力 Key 泄露、误删或触发额度后，先切换备用 Key，再处理旧配置。

备用 Key 最好也按用途区分，不要把一把“万能备用 Key”同时塞给所有工具。否则主力 Key 出问题时，备用 Key 很快又会变成新的单点。
