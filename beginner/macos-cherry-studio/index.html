macOS 接入 Cherry Studio ​

Cherry Studio 是图形界面聊天客户端，不需要安装命令行工具、Homebrew 或 Node.js。适合第一次验证接口是否可用。

前置条件

开始之前，确保你已经完成：

  1. ✅ 小蓝中转站 账号已注册并充值
  2. ✅ 已创建 API 密钥（以 sk- 开头的字符串）
  3. ✅ 已从模型广场复制模型名

还没做？先回 macOS 快速接入 的第一阶段。

第 1 步：下载安装 ​

从 Cherry Studio 官网下载页 下载 macOS 版本。

  • Apple 芯片（M1/M2/M3/M4）选 ARM64
  • Intel 芯片选 x64

不确定选哪个？点左上角 → 关于本机 查看芯片类型。

下载后打开 .dmg 文件，将 Cherry Studio 拖到「应用程序」文件夹。

macOS 提示「无法验证开发者」？

第一次打开时 macOS 可能拦截。处理方式：系统设置 → 隐私与安全性 → 找到 Cherry Studio → 点击 仍要打开。

第 2 步：添加 小蓝中转站 服务商 ​

打开 Cherry Studio 后，进入 设置（左下角齿轮图标）→ 服务商，点击添加新的服务商。

按以下信息填写：

字段填什么注意
服务商类型OpenAI 兼容不要选其他类型
API 地址https://xiaolan1.cn不要加 /v1，Cherry Studio 会自己拼接
API Keysk-你的专属Key从控制台复制的完整 Key

填好后点 保存。

第 3 步：验证连接 ​

保存后点 Check（或「检查」按钮），验证连接是否正常。

如果 Check 通过，点 拉取模型列表，Cherry Studio 会自动获取可用模型。

第 4 步：发送测试消息 ​

新建一个对话，在模型下拉框中选一个刚拉下来的模型，发送「你好」。

能收到回复，说明配置成功。

常见问题 ​

现象可能原因怎么处理
macOS 拦截安装安全策略系统设置 → 隐私与安全性 → 允许打开
Check 失败API 地址或 Key 错误确认地址是 https://xiaolan1.cn（不带 /v1），Key 以 sk- 开头
模型列表为空Key 没有模型权限回控制台检查 Key 的分组和模型权限
模型列表有但发消息报错模型选错或余额不足确认选的模型在列表中，检查余额

下一步 ​

Cherry Studio 配置成功后，说明你的 小蓝中转站 账号、Key 和余额都没问题。如果想继续接入命令行工具：

  • → Claude Code 接入（需要先做环境准备）
  • → Codex 接入（需要先做环境准备）
